  <name>Task 2: Create video probing utilities</name>
  <files>src/video_processing/__init__.py, src/video_processing/video_utils.py</files>
  <action>
1. Create `src/video_processing/` directory and `__init__.py`

2. Create `src/video_processing/video_utils.py` with these functions:

**probe_video(video_path: str | Path) -> dict:**
- Wraps `ffmpeg.probe()` with error handling
- Returns raw probe dict or raises descriptive error if file invalid/missing
- Catches `ffmpeg.Error` and re-raises with user-friendly message

**get_video_info(video_path: str | Path) -> VideoInfo:**
- Calls probe_video() and extracts structured info
- Returns dataclass/NamedTuple with: width, height, duration, codec, fps, container_format, has_audio
- Handles edge cases: missing audio stream, variable frame rate
- fps calculation: parse `r_frame_rate` string (e.g., "30000/1001" -> 29.97)

**detect_container_format(video_path: str | Path) -> str:**
- Returns normalized format string: "mp4", "mkv", or "avi"
- Maps FFmpeg format names: "mov,mp4,m4a,3gp,3g2,mj2" -> "mp4", "matroska,webm" -> "mkv", "avi" -> "avi"
- Raises ValueError for unsupported formats

**validate_video_file(video_path: str | Path) -> tuple[bool, str]:**
- Returns (True, "") if valid, (False, "error message") if invalid
- Checks: file exists, has video stream, supported format
- Uses probe_video() internally

Reference patterns from 02-RESEARCH.md Pattern 2 (Metadata Probing).

  <action>
1. Create `src/storage/` directory and `__init__.py`

2. Create `src/storage/temp_manager.py` with:

**TempFileManager class:**
- Context manager that wraps tempfile.TemporaryDirectory
- `__init__(base_dir: Path = None, prefix: str = "voicedub_")` - uses system temp by default
- `__enter__()` - creates temp directory, returns self
- `__exit__()` - cleans up directory and all contents
- Properties: `temp_dir: Path`, `audio_path: Path`, `frames_dir: Path`
- Creates subdirectories for audio and frames on enter

**create_temp_directory(prefix: str = "voicedub_") -> Path:**
- Convenience function for simple temp dir creation
- Returns Path to created directory
- Caller responsible for cleanup (for cases where context manager doesn't fit)

**get_temp_file_path(temp_dir: Path, filename: str) -> Path:**
- Returns path for a file within temp directory
- Creates parent directories if needed

Usage pattern (for future plans):
```python
with TempFileManager() as temp:
    audio_path = temp.audio_path  # Pre-created path for audio.wav
    frames_dir = temp.frames_dir  # Pre-created directory for frames
    # ... processing ...
# Automatic cleanup when exiting context
```

Reference patterns from 02-RESEARCH.md Pattern 3 (Temporary File Management).
