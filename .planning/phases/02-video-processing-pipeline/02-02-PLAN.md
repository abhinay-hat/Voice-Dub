  <name>Task 1: Create audio/video extractor module</name>
  <files>src/video_processing/extractor.py</files>
  <action>
Create `src/video_processing/extractor.py` with:

**extract_audio(video_path: Path, output_path: Path, sample_rate: int = 48000) -> Path:**
- Extracts audio stream to WAV format (PCM 16-bit signed little-endian)
- Parameters: video_path (input), output_path (where to save WAV), sample_rate (default 48kHz - video production standard)
- Uses ffmpeg-python: `ffmpeg.input(video_path).output(output_path, acodec='pcm_s16le', ar=sample_rate, ac=2)`
- Handles videos with no audio stream (raises descriptive error)
- Returns output_path on success
- Runs with `quiet=True` to suppress FFmpeg output, `overwrite_output=True` to replace existing

**extract_video_stream(video_path: Path, output_path: Path) -> Path:**
- Extracts video stream without audio (for later recombination)
- Uses stream copy (`vcodec='copy'`) to avoid re-encoding - fast and lossless
- Removes audio stream: `ffmpeg.input(video_path).output(output_path, vcodec='copy', an=None)` (an=None removes audio)
- Returns output_path on success

**ExtractionResult dataclass:**
- audio_path: Path (extracted WAV)
- video_path: Path (video-only file)
- duration: float (seconds)
- sample_rate: int
- video_info: VideoInfo (from video_utils)

**extract_streams(video_path: Path, temp_dir: Path) -> ExtractionResult:**
- Convenience function that extracts both audio and video
- Uses get_video_info() to populate metadata
- Returns ExtractionResult with all paths and metadata

Reference patterns from 02-RESEARCH.md Pattern 1 (Stream Copy Extraction).

Error handling:
- Catch `ffmpeg.Error` and parse stderr for user-friendly messages
- Validate input file exists before processing
- Check for audio stream presence before audio extraction

<task type="auto">
  <name>Task 2: Create audio-video merger module</name>
  <files>src/video_processing/merger.py</files>
  <action>
Create `src/video_processing/merger.py` with:

**MergeConfig dataclass:**
- video_codec: str = "copy" (stream copy by default - fast, lossless)
- audio_codec: str = "aac" (for MP4 compatibility) or "copy" for MKV
- audio_bitrate: str = "192k" (standard quality)
- container_format: str = None (auto-detect from output path)

**merge_audio_video(video_path: Path, audio_path: Path, output_path: Path, config: MergeConfig = None) -> Path:**
- Merges video stream with new/processed audio stream
- Uses ffmpeg with explicit stream mapping
- Video input: `ffmpeg.input(video_path).video`
- Audio input: `ffmpeg.input(audio_path).audio`
- Combines: `ffmpeg.output(video_in, audio_in, output_path, vcodec=config.video_codec, acodec=config.audio_codec)`
- Handles container format: MP4 needs AAC audio, MKV can use copy
- Returns output_path on success

**get_optimal_merge_config(input_format: str, output_format: str) -> MergeConfig:**
- Returns optimal config based on container formats
- MP4 output: video copy, audio AAC (required for compatibility)
- MKV output: both copy (MKV supports almost any codec)
- AVI output: video copy, audio mp3 (for compatibility)

**validate_merge_inputs(video_path: Path, audio_path: Path) -> tuple[bool, str]:**
- Checks both files exist and are valid
- Verifies video has video stream, audio has audio stream
- Returns (True, "") or (False, "error message")

Reference patterns from 02-RESEARCH.md Pattern 4 (Stream Merging with Synchronization).

Container format decisions from 02-CONTEXT.md:
- Match input format (MKV in -> MKV out)
- User decision: Claude chooses codecs, so select based on compatibility

**ProcessingResult dataclass:**

**ProcessingProgress callback protocol:**
- Callable[[float, str], None] - (progress 0.0-1.0, description)
- Optional callback for progress updates

**process_video(input_path: Path, output_path: Path = None, progress_callback: Callable = None) -> ProcessingResult:**
Main orchestration function:

1. Validate input video (using validate_video_file)
2. Detect input format (using detect_container_format)
3. Determine output path (if None, use input name + "_processed" + same extension)
4. Report progress: (0.0, "Starting processing")

5. Create TempFileManager context:
   ```python
   with TempFileManager() as temp:
       # Extract streams
       progress_callback(0.2, "Extracting audio")
       extraction = extract_streams(input_path, temp.temp_dir)

       # In Phase 2, audio is NOT processed - just passed through
       # Future phases will insert ML processing here
       processed_audio = extraction.audio_path

       progress_callback(0.6, "Merging streams")
       # Merge back
       merge_config = get_optimal_merge_config(input_format, output_format)
       merge_audio_video(extraction.video_path, processed_audio, output_path, merge_config)
   ```
   # Temp files automatically cleaned up here

6. Report progress: (1.0, "Complete")
7. Return ProcessingResult

**validate_processing_environment() -> tuple[bool, str]:**
- Checks FFmpeg is available
- Verifies temp directory is writable
- Returns (True, "") or (False, "error message")

Integration:
- Import from video_utils (get_video_info, detect_container_format, validate_video_file)
- Import from extractor (extract_streams)
- Import from merger (merge_audio_video, get_optimal_merge_config)
- Import from temp_manager (TempFileManager)

This pipeline does a "round trip" (extract -> pass through -> merge) to validate the toolchain. Future phases will insert ML processing between extraction and merging.
