  <action>
Two files need coordinated updates:

**src/tts/speaker_embeddings.py:**

1. Update `put()` to always store tensors on CPU regardless of input device:
   ```python
   def put(self, speaker_id, gpt_cond_latent, speaker_embedding):
       # Always store on CPU to minimize VRAM when multiple models co-resident
       self._cache[speaker_id] = (
           gpt_cond_latent.cpu().clone(),
           speaker_embedding.cpu().clone()
       )
       self._locations[speaker_id] = 'cpu'
   ```
   Use `.cpu().clone()` to detach from any computation graph and ensure we own the data.

2. Add a new method `get_on_device(speaker_id, device='cuda')` that returns tensors moved to the specified device WITHOUT modifying the cache:
   ```python
   def get_on_device(self, speaker_id: str, device: str = 'cuda') -> Optional[Tuple[torch.Tensor, torch.Tensor]]:
       """Return embeddings moved to device. Cache remains on CPU."""
       entry = self._cache.get(speaker_id)
       if entry is None:
           return None
       gpt_cond_latent, speaker_embedding = entry
       return (gpt_cond_latent.to(device), speaker_embedding.to(device))
   ```
   This avoids mutating cache state when synthesizing -- important because the cache is shared across all segments.

3. Keep existing `get()` method unchanged (returns CPU tensors). Keep `move_to_gpu()` and `move_to_cpu()` for backward compatibility.

**src/tts/xtts_generator.py:**

4. Update `synthesize_segment()` to use `get_on_device()` instead of `get()`:
   ```python
   result = self.embedding_cache.get_on_device(speaker_id, device='cuda')
   if result is None:
       raise ValueError(f"Speaker {speaker_id} not in embedding cache")
   gpt_cond_latent, speaker_embedding = result
   ```
   This ensures latents are on GPU for inference while cache stays on CPU.

5. Update `_synthesize_with_speed()` the same way -- use `get_on_device()` instead of `get()`.

6. No changes to `synthesize_all_segments()` or `synthesize_with_duration_matching()` -- they call synthesize_segment/_synthesize_with_speed which handle the GPU transfer.

IMPORTANT: Do NOT change the `generate_speaker_embeddings()` function or `generate_single_embedding()` -- they create embeddings which go through `put()`, and `put()` now handles the CPU storage automatically.
