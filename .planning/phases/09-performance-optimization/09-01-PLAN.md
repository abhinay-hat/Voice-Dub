<objective>
Upgrade the ModelManager from single-slot sequential loading to LRU multi-slot management, and create performance benchmarking utilities for GPU timing and utilization measurement.

Purpose: The current ModelManager (src/models/model_manager.py) unloads one model before loading another. With 32GB VRAM and main-process models totaling ~11GB (Whisper 4.5GB + SeamlessM4T 2.5GB + XTTS 4GB), all three can be co-resident. LRU multi-slot eliminates unnecessary unload/reload cycles, saving minutes per pipeline run. The PerformanceBenchmarker provides the instrumentation needed to validate the Phase 9 success criterion of >80% GPU utilization.

Output: Updated model_manager.py with LRU logic, new performance_benchmarker.py utility, and unit tests validating eviction behavior.

  <name>Task 1: Upgrade ModelManager to LRU Multi-Slot</name>
  <files>src/models/model_manager.py</files>
  <action>
Rewrite ModelManager to support multiple co-resident models using `collections.OrderedDict` for LRU tracking. Key changes:

1. Replace `self.current_model` / `self.current_model_name` with `self._models: OrderedDict[str, Any]` that maps model_name -> model_instance.
2. Add `max_slots: int = 3` constructor parameter (default 3 for Whisper+SeamlessM4T+XTTS).
3. Update `load_model(name, loader_fn)`:
   - If `name` already in `_models`: call `self._models.move_to_end(name)` to mark as recently used, return existing model. Do NOT reload.
   - If `len(self._models) >= max_slots`: evict LRU entry via `self._models.popitem(last=False)` and run 4-step cleanup on evicted model (move to CPU, del, gc.collect, torch.cuda.empty_cache).
   - Otherwise: call `loader_fn()`, store in `self._models[name]`.
   - Return the model.
4. Update `unload_current_model()` to `unload_model(name: Optional[str] = None)`:
   - If name provided: unload that specific model.
   - If name is None: unload ALL models (iterate and cleanup each).
   - Keep backward compatibility: `unload_current_model()` calls `unload_model()` with no args.
5. Update `get_current_model_name()` to `get_loaded_models() -> List[str]` returning list of loaded model names.
   - Keep `get_current_model_name()` as backward-compatible wrapper returning the most recently loaded model name (last key in OrderedDict) or None.
6. `_unload_model(name, model)` internal method: same 4-step cleanup (move to CPU, del ref, gc.collect, torch.cuda.empty_cache), plus verbose VRAM logging.
7. `__del__` calls `unload_model()` to clean up all models.

CRITICAL backward compatibility: The existing API signature `load_model(model_name, model_loader_fn)` and `unload_current_model()` MUST remain functional. All existing stage code (asr_stage.py, tts_stage.py, translation_stage.py) calls these methods and must continue working without changes. The only behavioral difference is that models are cached rather than evicted immediately.

   ```python
   class StageTimer:
       def __init__(self, stage_name: str):
           self.stage_name = stage_name
           self.elapsed_ms = 0.0
       def __enter__(self):
           # Use torch.cuda.Event(enable_timing=True) for GPU-accurate timing
           # torch.cuda.synchronize() before starting
           self._start_event = torch.cuda.Event(enable_timing=True)
           self._end_event = torch.cuda.Event(enable_timing=True)
           torch.cuda.synchronize()
           self._start_event.record()
           return self
       def __exit__(self, *exc):
           self._end_event.record()
           torch.cuda.synchronize()
           self.elapsed_ms = self._start_event.elapsed_time(self._end_event)
           return False
       @property
       def elapsed_seconds(self) -> float:
           return self.elapsed_ms / 1000.0
   ```
