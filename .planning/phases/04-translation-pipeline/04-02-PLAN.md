**Key functions:**

1. **estimate_duration(text: str, chars_per_second: float = None) -> float**
   - Normalize text (lowercase, collapse whitespace)
   - Remove spaces for character count (spaces don't contribute to speech duration)
   - Calculate: `char_count / chars_per_second`
   - Default chars_per_second from settings.TRANSLATION_CHARS_PER_SECOND (15)
   - Return estimated duration in seconds

2. **normalize_text_for_duration(text: str) -> str**
   - Convert to lowercase
   - Collapse multiple spaces: `" ".join(text.split())`
   - Remove leading/trailing whitespace
   - Return normalized text
   - Note: Future enhancement could expand numbers ("2026" -> "two thousand twenty-six"), but skip for now per speed-first priority

   - Estimate translated text duration using estimate_duration()
   - Default tolerance from settings.TRANSLATION_DURATION_TOLERANCE (0.1 = ±10%)
   - Calculate duration ratio: `estimated / original`
   - Check if within tolerance: `(1 - tolerance) <= ratio <= (1 + tolerance)`
   - Return dict with keys:
     ```python
     {
         "is_valid": bool,
         "estimated_duration": float,
         "original_duration": float,
         "ratio": float,  # estimated/original
         "tolerance": float,
         "min_ratio": float,  # 1 - tolerance
         "max_ratio": float,  # 1 + tolerance
     }
     ```

Create src/stages/translation/candidate_ranker.py with multi-candidate ranking logic.

**CandidateRanker class:**

Structure:
- `__init__(self, confidence_weight=0.6, duration_weight=0.4, chars_per_second=None)`
  - Store weights (must sum to 1.0, validate in init)
  - Default chars_per_second from settings
  - Weights reflect user priority: speed first (confidence) with duration fit as secondary concern

Methods:

1. **rank_candidates(candidates: list[str], scores: list[float], original_duration: float) -> tuple[dict, list[dict]]**
   - For each candidate, calculate:
     - Model confidence score (provided in scores list)
     - Duration fit score using _calculate_duration_score()
     - Combined weighted score: `confidence * confidence_weight + duration_fit * duration_weight`
   - Sort candidates by combined_score (descending)
   - Return: `(best_candidate_dict, all_ranked_list)`
   - Each dict contains:
     ```python
     {
         "candidate": str,
         "model_confidence": float,
         "duration_score": float,
         "combined_score": float,
         "estimated_duration": float,
         "duration_ratio": float,
         "is_valid_duration": bool
     }
     ```

2. **_calculate_duration_score(translated_text: str, original_duration: float, tolerance: float = 0.1) -> float**
   - Estimate duration using duration_validator.estimate_duration()
   - Calculate ratio: `estimated / original`
   - If within tolerance (0.9 <= ratio <= 1.1):
     - Return linear penalty: `1.0 - abs(1.0 - ratio) * 10`
     - Example: ratio=1.05 (5% over) -> score = 1.0 - 0.05*10 = 0.5
     - Example: ratio=1.0 (perfect) -> score = 1.0
   - If outside tolerance:
     - Return 0.0 (hard penalty for violating constraints)

**Helper function (module-level):**

**rank_candidates(candidates: list[str], scores: list[float], original_duration: float, confidence_weight=0.6, duration_weight=0.4) -> tuple[dict, list[dict]]**
- Convenience function that creates CandidateRanker and calls rank_candidates()
- Allows one-line usage without instantiating class

**Implementation requirements:**
- Import from .duration_validator: estimate_duration
- Import from src.config.settings: TRANSLATION_CHARS_PER_SECOND, TRANSLATION_DURATION_TOLERANCE
- Validate inputs: len(candidates) == len(scores), weights sum to 1.0
- Handle edge cases: empty candidates list (return None, []), single candidate (return it)
