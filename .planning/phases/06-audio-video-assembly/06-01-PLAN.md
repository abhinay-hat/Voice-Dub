**TimedSegment dataclass:**
```python
@dataclass
class TimedSegment:
    """Audio segment with high-precision timestamps."""
    start: float  # Float64 seconds
    end: float    # Float64 seconds
    audio_path: str
    speaker_id: str

    @property
    def duration(self) -> float:
        """Duration in seconds (float64 precision)."""
        return np.float64(self.end) - np.float64(self.start)

    def to_frame_boundary(self, fps: float = 30.0) -> tuple[float, float]:
        """Align timestamps to nearest frame boundaries for fps."""
        frame_duration = np.float64(1.0) / np.float64(fps)
        start_frame = np.round(self.start / frame_duration)
        end_frame = np.round(self.end / frame_duration)
        return (float(start_frame * frame_duration), float(end_frame * frame_duration))
```

**validate_timestamps_precision():**
- Takes List[TimedSegment] or List[dict with start/end keys]
- Validates all timestamps are float (Python float is 64-bit)
- Validates no timestamps are negative
- Validates end > start for each segment
- Validates no gaps > 1 second between consecutive segments (optional, warn)
- Raises ValueError with descriptive message if validation fails
- Returns True if all validations pass

**ensure_float64():**
- Utility to convert any numeric timestamp to np.float64
- Handles int, float, np.float32 inputs
- Returns np.float64

Include docstrings with examples and rationale for float64 (prevents precision loss over 20min, research pitfall #2).

Create two modules for audio processing:

**src/assembly/audio_normalizer.py:**

**normalize_sample_rate():**
- Takes audio_path (Path), target_sr (int, default 48000), quality (str, default 'kaiser_best')
- Loads audio with librosa.load(sr=None) to preserve original sample rate
- If already at target_sr, returns original path (no processing)
- Resamples using librosa.resample with res_type=quality
- Writes to {stem}_48k.wav using soundfile.write with PCM_16 subtype
- Returns Path to normalized file
- Log message showing: original_sr -> target_sr conversion

**validate_sample_rate():**
- Takes audio_path, expected_sr (default 48000)
- Uses soundfile.info() to check actual sample rate
- Returns tuple[bool, int] - (matches_expected, actual_sample_rate)

**batch_normalize():**
- Takes List[Path], target_sr, quality
- Normalizes all files, returns List[Path] with normalized paths
- Handles mixed sample rates in batch

**src/assembly/audio_concatenator.py:**

**concatenate_audio_segments():**
- Takes List[TimedSegment] (already normalized to 48kHz)
- Validates all segments have same sample rate (use validate_sample_rate)
- Loads each segment with librosa, concatenates using np.concatenate
- Handles gaps between segments: insert silence padding at 48kHz
- Writes concatenated audio to output_path
- Returns Path to concatenated file

**get_total_duration():**
- Takes List[TimedSegment]
- Returns total duration as float64 (sum of all durations)

Key implementation notes:
- Use librosa.resample res_type='kaiser_best' (not 'scipy' or 'linear') per research
- Use soundfile for I/O (preserves metadata, already in project)
- Handle mono/stereo: ensure output is mono (for TTS compatibility)
- PCM_16 subtype for broad compatibility
