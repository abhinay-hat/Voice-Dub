**SyncCheckpoint dataclass:**
```python
@dataclass
class SyncCheckpoint:
    """Validation point for audio-video synchronization."""
    timestamp: float      # Expected timestamp (seconds)
    frame_number: int     # Expected video frame at this point
    audio_sample: int     # Expected audio sample index
    expected_duration: float  # Expected cumulative duration at checkpoint
    actual_duration: float    # Actual cumulative duration from segments
    drift_ms: float       # Measured drift (actual - expected) in milliseconds
    within_tolerance: bool    # True if abs(drift_ms) <= tolerance
```

**DriftValidationResult dataclass:**
```python
@dataclass
class DriftValidationResult:
    """Complete drift validation result."""
    is_synced: bool           # True if all checkpoints pass
    checkpoints: List[SyncCheckpoint]
    max_drift_ms: float       # Maximum absolute drift observed
    total_duration: float     # Total video duration validated
    checkpoint_count: int     # Number of checkpoints checked
```

**validate_sync_at_intervals():**
- Takes: segments (List[TimedSegment]), video_fps (float), audio_sr (int), total_duration (float), interval_seconds (float, default 300.0), tolerance_ms (float, default 45.0)
- Generates checkpoints at interval_seconds (e.g., 300, 600, 900, 1200 for 20-min video)
- Also adds checkpoint at total_duration (video end)
- For each checkpoint:
  - Calculate expected cumulative duration at that timestamp
  - Calculate actual cumulative duration from segment list
  - drift_ms = (actual - expected) * 1000
  - within_tolerance = abs(drift_ms) <= tolerance_ms
  - Calculate frame_number = int(timestamp * video_fps)
  - Calculate audio_sample = int(timestamp * audio_sr)
- Returns DriftValidationResult with all checkpoint data
- Does NOT raise exception - caller decides how to handle drift

**check_segment_continuity():**
- Takes: segments (List[TimedSegment])
- Validates segments are in chronological order (start time increases)
- Validates no overlapping segments
- Validates gaps < 1 second (warn, don't fail)
- Returns tuple[bool, List[str]] - (is_continuous, warning_messages)

Implementation notes:
- Use float64 arithmetic (np.float64) for all timestamp calculations
- Log checkpoint results for debugging
- Success criteria requires checking at 5-min, 10-min, 15-min, 20-min marks
- 45ms tolerance is ATSC recommendation (research)

**MergeResult dataclass:**
```python
@dataclass
class MergeResult:
    """Result of audio-video merge operation."""
    output_path: Path
    video_codec: str      # e.g., 'copy'
    audio_codec: str      # e.g., 'aac'
    async_correction: bool  # Whether aresample async=1 was applied
    merge_duration: float   # Time to complete merge (seconds)
```

**merge_with_sync_validation():**
- Takes: video_path (Path), audio_path (Path), output_path (Path), async_correction (bool, default True)
- Uses ffmpeg-python to build merge pipeline:
  1. Create video_input = ffmpeg.input(str(video_path))
  2. Create audio_input = ffmpeg.input(str(audio_path))
  3. If async_correction: apply audio_stream.filter('aresample', async=1)
  4. Explicit stream mapping: video_input.video, audio_stream
  5. Output options:
     - vcodec='copy' (no video re-encoding)
     - acodec='aac' for MP4, 'copy' for MKV (detect from output extension)
     - audio_bitrate='192k' (standard quality)
     - map_metadata=0 (preserve metadata)
     - movflags='faststart' (MP4 streaming optimization)
- Validates inputs exist before merge (use Phase 2 validate_merge_inputs pattern)
- Runs ffmpeg.run(stream, quiet=True, overwrite_output=True)
- Returns MergeResult with merge details

**get_merge_config():**
- Takes output_format (str)
- Returns dict with optimal codec settings per format
- MP4: {'vcodec': 'copy', 'acodec': 'aac', 'audio_bitrate': '192k'}
- MKV: {'vcodec': 'copy', 'acodec': 'copy'}
- AVI: {'vcodec': 'copy', 'acodec': 'mp3', 'audio_bitrate': '192k'}

**validate_audio_video_compatibility():**
- Takes video_path, audio_path
- Uses ffprobe to check:
  - Video has video stream
  - Audio has audio stream
  - Audio sample rate is 48kHz (warn if not)
- Returns tuple[bool, List[str]] - (is_compatible, warnings)

Implementation notes:
- The key difference from Phase 2 merger.py is the `-af aresample=async=1` filter
- This replaces deprecated `-async 1` flag per research findings
- async=1 stretches/compresses audio to match video clock (prevents drift)
- Use explicit stream mapping (not auto-selection)
- Handle ffmpeg.Error with descriptive error messages
