2. Implement reference_extractor.py with:

**ReferenceExtractor class:**
- Constructor takes audio_path (Path) and sample_rate (default 24000 for XTTS)
- Loads audio lazily on first access using librosa

**select_best_segment(speaker_segments: list, min_duration: float = 6.0, max_duration: float = 10.0) -> dict:**
- Takes list of segment dicts with keys: speaker, start, end, duration
- Filters segments >= min_duration
- Calculates RMS energy for each candidate (proxy for audio clarity)
- Returns segment dict with highest RMS energy
- If no single segment >= min_duration: implement concatenation fallback (combine adjacent same-speaker segments until reaching min_duration)
- If concatenation still insufficient: return None (caller handles fallback)

**extract_reference_samples(translation_json: Path, audio_path: Path, output_dir: Path) -> dict[str, Path]:**
- Load translation JSON to get speaker IDs and segment timestamps
- Group segments by speaker_id
- For each unique speaker:
  - Call select_best_segment() with that speaker's segments
  - Extract audio segment using librosa (start_sample to end_sample)
  - If segment > max_duration: extract center 10s portion
  - Save to output_dir/reference_{speaker_id}.wav at 24kHz (XTTS native)
  - Use soundfile for writing WAV (better codec handling than librosa)
- Return dict mapping speaker_id -> reference_audio_path
- Handle edge case: speaker with no viable reference (log warning, return None for that speaker)

**Critical implementation details:**
- Use librosa.load() with sr=24000 (XTTS native sample rate)
- RMS energy calculation: np.sqrt(np.mean(audio_segment**2))
- For concatenation: ensure gap between segments < 0.5s to avoid unnatural pauses
- Validate audio is mono (XTTS requirement)

- Methods: get(speaker_id), put(speaker_id, embeddings), has(speaker_id), move_to_cpu(speaker_id), clear()

**generate_speaker_embeddings(xtts_model, reference_paths: dict[str, Path]) -> SpeakerEmbeddingCache:**
- Takes loaded XTTS model instance and dict of speaker_id -> reference_audio_path
- For each speaker:
  - Call model.get_conditioning_latents(audio_path=str(reference_path))
  - This returns (gpt_cond_latent, speaker_embedding) tuple
  - Store in SpeakerEmbeddingCache
- Return populated cache

**generate_single_embedding(xtts_model, audio_path: Path) -> tuple:**
- Thin wrapper around model.get_conditioning_latents()
- Validates audio file exists and is readable
- Returns (gpt_cond_latent, speaker_embedding) tuple

**Critical implementation details:**
- XTTS model's get_conditioning_latents() accepts audio_path as string
- Embeddings are GPU tensors by default; cache tracks location
- VRAM management: if processing many speakers, move completed embeddings to CPU
- Use torch.cuda.empty_cache() after moving embeddings to CPU
- Error handling: if embedding generation fails, log error and return None for that speaker
